## 🎯 THE FIVE PILLARS IN DETAIL

### PILLAR 1: ZPWO (Zero-Point Workflow Orchestrator)

**Role:** Master conductor at FLOW center
**Question:** "What phase are we in?"

**Responsibilities:**
- Phase orchestration (P1 → P2 → P3 → **P4 NEW**)
- SSOT locking (Stage A creates, Stage B consumes)
- Session state tracking
- Circuit breaker enforcement (max 3 FIX loops)
- Garbage collection at phase transitions

**Delegates To:**
- TWA: "Where does this task go?"
- TWE: "What tools do we need?"
- MMA: "Is this quality acceptable?"
- **Deconstructor: "What can we learn from this?"** ← NEW

**Zero-Point Discipline:** Returns to minimal context after each phase

---

### PILLAR 2: TWA (Tools & Workflow Architect)

**Role:** Real-time router and pipeline manager
**Question:** "Where does this task go?"

**Three Engines:**
1. **Skill Orchestration** — Route task → correct skill
2. **Tool Coordination** — Select minimal tools (Skills > MCPs)
3. **Pipeline Management** — Multi-step workflow state

**Bottleneck Diagnosis:**
| Bottleneck | Resolution |
|------------|------------|
| Skill Gap | Check manifest, build if needed |
| Tool Gap | Consult TWE, prefer scripts |
| Pipeline Stuck | Break cycles, check state |
| Quality Loop | Route to specialist or Deconstructor |
| Context Bloat | Trigger Zero-Point reset |

**MMA Result Routing:**
```
FULL PASS (≥8.5)     → Next step or complete
CONDITIONAL (≥7.0)   → Polish loop (MWP)
FAIL (<7.0)          → Specialist (HPE/NRA) or Deconstructor for diagnosis
HARD FAIL            → Escalate to human
```

**NEW - Deconstructor Routing:**
```
High performer (≥8.5) → Deconstruct to extract patterns
Underperformer (<7.0) → Deconstruct to diagnose gaps
A/B test complete    → Deconstruct both to learn winner
```

---

### PILLAR 3: TWE (Tools & Workflow Engineer)

**Role:** Toolchain planner and execution designer
**Question:** "What tools do we use?"

**Core Principles:**
- **Zero-Point:** Keep FLOW center light
- **Skills > MCPs:** Prefer on-demand activation over always-on
- **Heal & Renew:** Every run outputs correction log + patch proposals

**Outputs:**
- WORKFLOW_PLAN (what steps, what order)
- TOOLCHAIN_PLAN (which tools for which steps)
- RUNBOOK (reproducible instructions)
- SCRIPT_CALL_SPECS (exact tool invocations)
- **CORRECTION_LOG** (what went wrong)
- **PATCH_PROPOSALS** (how to improve)

**Integration with Deconstructor:**
TWE's PATCH_PROPOSALS + Deconstructor's PATCH_PROPOSALS → Combined improvement pipeline

---

### PILLAR 4: MMA (Master Monitor Agent)

**Role:** Quality guardian and 7D validator
**Question:** "Is this good enough?"

**The 7 Dimensions:**
1. **Strategy Alignment** (≥8.0) - Matches PROJECT_BRIEF/MESSAGE_SPINE
2. **Proof Discipline** (≥9.0 CRITICAL) - Claims grounded in EVIDENCE_PACK
3. **CTA Integrity** (≥9.0 CRITICAL) - Single CTA, no fake urgency
4. **Voice Consistency** (≥8.0) - Matches VOICE_GUIDE
5. **Clarity + Structure** (≥8.0) - Clear, scannable, mobile-friendly
6. **Resonance** (≥8.0) - Emotionally engaging
7. **Ethical Guardrails** (≥9.0 CRITICAL) - No manipulation, compliant

**Verdicts:**
```
PASS (all dimensions ≥threshold, avg ≥8.0) → Advance
FIX (fixable issues) → Loop (max 3x)
ESCALATE (critical fail or 3 loops exceeded) → Human decision
```

**Integration with Deconstructor:**
MMA scores become input for Deconstructor analysis (what scored high/low, why?)

---

### PILLAR 5: DECONSTRUCTOR (Sales Page Deconstructor Analyst) ⭐ NEW

**Role:** Learning engine and pattern extractor
**Question:** "What can we learn from this?"

**Core Capabilities:**

**1. Structure Extraction**
- Identify sections (headline, lead, mechanism, proof, offer, CTA)
- Map flow sequence
- Measure transitions
- Extract CTA positioning

**2. Mechanism Analysis**
- Extract promise (primary, emotional, tangible, timeframe)
- Identify mechanism (name, category, explanation, unique element)
- Map vs. alternatives (why they fail)
- Extract belief shifts

**3. Offer Deconstruction**
- Extract value equation (Dream Outcome, Likelihood, Time, Effort)
- Map value stack (core + bonuses)
- Analyze pricing psychology
- Document CTA structure

**4. Voice Signature Analysis**
- Tone markers (formality, warmth, authority, urgency)
- Sentence rhythm (short/medium/long ratio)
- Word choice patterns (second-person, active voice, sensory words)
- Power phrases used / forbidden words avoided

**5. 7S/7F Coverage Audit**
- Score each dimension (SAFE, SPECIAL, SMART, SIGNIFICANT, SUPPORTED, SUPERIOR, STEAL)
- Map to Neuro-Box 6D axes
- Identify gaps and balance

**6. Pattern Recognition**
- Match against Pattern Library
- Flag new patterns
- Confirm existing patterns
- Identify anti-patterns

**7. Patch Proposal Generation**
- Identify skill gaps
- Propose specific improvements
- Prioritize (HIGH/MEDIUM/LOW)
- Estimate implementation effort

**Outputs:**
- DECONSTRUCTION_REPORT (full analysis)
- PATTERN_LIBRARY_UPDATE (new patterns discovered)
- PATCH_PROPOSALS (skill improvements)
- LEARNING_FRAMEWORK (meta-insights)

**Integration Points:**
- **ZPWO:** Triggered as Phase 4 (post-production learning)
- **TWA:** Routes high/low performers to Deconstructor
- **TWE:** Receives Deconstructor's patch proposals
- **MMA:** Provides quality scores as input for analysis
- **Skills:** Receive updates from approved patches

## 🔗 INTEGRATION PATTERNS

### Pattern 1: Standard Production (P1 → P2 → P3)

**No Deconstructor involved yet**

```
User Request → ZPWO → TWA routes → TWE plans → Skill executes
                ↓
              MMA validates → PASS/FIX/ESCALATE
                ↓
              ZPWO manages loop → Final asset
```

**Used when:** Creating new assets, business as usual

---

### Pattern 2: Production + Learning (P1 → P2 → P3 → P4)

**Deconstructor added for continuous improvement**

```
User Request → ZPWO → TWA → TWE → Skill → MMA → Asset
                                              ↓
                                    Asset deployed + performs
                                              ↓
                          ZPWO activates P4 (Learning)
                                              ↓
                          Deconstructor analyzes asset
                                              ↓
                          Pattern Library updated
                                              ↓
                          Patch proposals generated
                                              ↓
                          **Human approves patches**
                                              ↓
                          TWE applies patches to skills
                                              ↓
                          Skills improved (v[X+1])
```

**Used when:** Want to learn from what was created

---

### Pattern 3: Performance Analysis (Deconstructor-first)

**Start with analysis, then improve**

```
Existing asset (yours or competitor) → Deconstructor
                                              ↓
                          DECONSTRUCTION_REPORT generated
                                              ↓
                          Patterns extracted
                                              ↓
                          **Human reviews learnings**
                                              ↓
                          Patterns added to library
                                              ↓
                          Next ZPWO run uses updated patterns
```

**Used when:** Learning from existing work before creating new

---

### Pattern 4: A/B Test Learning

**Compare two versions to learn what worked**

```
Asset A + Asset B (A/B test complete with winner data)
                                              ↓
                          Deconstructor analyzes BOTH
                                              ↓
                          Comparative analysis: What differed?
                                              ↓
                          Hypothesis: Why A won vs. B (or vice versa)
                                              ↓
                          Pattern extracted from winner
                                              ↓
                          **Human validates hypothesis**
                                              ↓
                          Pattern added to library
                                              ↓
                          Future assets leverage validated pattern
```

**Used when:** Have performance data on multiple versions

---

### Pattern 5: Diagnostic Deep-Dive

**Page underperformed, need to understand why**

```
Underperforming asset → Deconstructor
                                              ↓
                          Full 7S/7F audit
                                              ↓
                          Compare to high-performing benchmarks
                                              ↓
                          Identify gaps (missing dimensions, weak mechanism)
                                              ↓
                          Generate specific fixes
                                              ↓
                          **Human reviews diagnosis**
                                              ↓
                          Rebuild asset with Deconstructor insights
                                              ↓
                          OR: Create patch to prevent same issue
```

**Used when:** Something failed and you need to know why
