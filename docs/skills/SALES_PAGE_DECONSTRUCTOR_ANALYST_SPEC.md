## 🎯 CORE PURPOSE

### What It Does
**Reverse-engineers successful sales pages** to extract:
1. **Structure patterns** (flow, sections, sequence)
2. **Mechanism frameworks** (how persuasion works)
3. **Offer architectures** (value stack, pricing psychology)
4. **Voice signatures** (tone, rhythm, style markers)
5. **7S/7F coverage** (resonance dimension mapping)
6. **Learning frameworks** (what works, why, when)
7. **Patch proposals** (how to improve our skills)

### Why It's Critical
**Self-healing architecture requires learning loops:**
- Analyze what resonates → Extract patterns → Update skills → Generate better → Repeat
- Human review checkpoints ensure quality control
- System improves from real-world feedback

## 🎯 USE CASES

### Use Case 1: Learn from Your Own Best Work
**Scenario:** You created sales pages with Claude that "came out pretty nice"

**Workflow:**
1. Feed your best pages to Deconstructor
2. Extract what made them work
3. Generate patches to codify patterns
4. Update skills with learned patterns
5. Future pages leverage what worked

**Benefit:** Your successes become templates for future work

---

### Use Case 2: Competitive Analysis
**Scenario:** Competitor has high-converting page in your market

**Workflow:**
1. Deconstruct competitor page
2. Extract mechanism framework
3. Identify what resonates with avatar
4. Generate alternative angles (not copy)
5. Update MESSAGE_SPINE with insights

**Benefit:** Learn market-tested patterns without copying

---

### Use Case 3: A/B Test Learning
**Scenario:** Tested two versions, one outperformed

**Workflow:**
1. Deconstruct both versions
2. Compare structure, voice, offer
3. Identify winning elements
4. Generate hypothesis (why it won)
5. Patch skills with validated patterns

**Benefit:** A/B test results improve system, not just one page

---

### Use Case 4: Multi-Market Pattern Extraction
**Scenario:** Multiple successful pages across different markets

**Workflow:**
1. Deconstruct pages from 5+ markets
2. Extract universal patterns
3. Identify market-specific patterns
4. Generate meta-framework
5. Update skills with universal + conditional logic

**Benefit:** Build cross-market intelligence

---

### Use Case 5: Performance Post-Mortem
**Scenario:** Page underperformed, need diagnosis

**Workflow:**
1. Deconstruct underperforming page
2. Compare to high-performing benchmarks
3. Identify gaps (missing 7S dimensions, weak mechanism, etc.)
4. Generate specific fixes
5. Rebuild with Deconstructor insights

**Benefit:** Learn from failures as much as successes
